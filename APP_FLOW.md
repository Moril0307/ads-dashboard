AdsDataHub - 应用交互流程
1. 数据准备阶段
指标表上传（CSV 1）: 用户上传包含“多日期、多系列”的广告数据表，列包含【日期/天、广告系列、预算、费用】。系统支持一次性导入过去 7-30 天的历史指标数据。

新 JID 付费表上传（CSV 2）: 用户上传包含“多日期”的新 JID 付费表，列包含【日期、广告系列、新 JID 付费数量】。系统支持一次性导入过去一段时间的历史付费人数数据。

2. 数据处理流
时间轴对齐: 系统按“日期 + 广告系列(campaign)”对齐两张 CSV 的数据，将指标表中的【预算、费用】与新 JID 付费表中的【新 JID 付费数量】合并在一起。

动态汇总: 根据用户选中的日期范围（Date Range），自动计算该时段内的总消耗、总新 JID 付费人数和平均新用户 CPA，其中：
- 新用户 CPA = 费用 / 新 JID 付费人数（若人数为 0 或缺失，则 CPA 显示为 -）。

3. 看板展示流（产品透视表模式 · Pivot View）

产品页签 (Product Tabs): 顶部使用 Tabs 切换 Fachat / Parau / Pinkpinkchat。切换后仅加载该产品前缀（ft / pu / ppt）对应的广告系列数据。

日期与汇总: 提供日期选择器（Date Picker），仅显示用户选定且存在数据的日期，不自动补全空白日期。在每个 Tab 的表格顶部保留该产品线在该时段的总消耗、总付费、平均 CPA 汇总卡片。

透视表布局 (Pivot View):
- 纵轴 (Y): 左侧首列为「日期」，在横向滚动时保持 Sticky 固定。
- 横轴 (X): 广告系列（Campaign）横向排布；每个 Campaign 为一个「列组」，包含子列：预算、消耗、付费、新 JID 用户数、新 JID CPA。
- 交互: 左侧日期列固定，右侧所有 Campaign 列组支持横向滚动（overflow-x-auto + CSS sticky）。

实现: 使用 Table 组件配合 CSS sticky 与 overflow-x-auto 实现；B–G 列展示一个 Campaign 的多维数据，H–M 列展示下一个，依此类推。
